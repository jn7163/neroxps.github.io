免流原理分析

免流我的理解，其实就是透过http代理，将流量伪装成发往运营商免费流量的目的地，再通过代理服务器上网从而达到免流的目的。

无论使用openvpn,还是HTTP Injector,都必须使用http代理,通常使用squid（因为猫腻就在http代理）.但是我这里介绍的方式是用mproxy.

mproxy :用C语言实现的一个http代理服务器,代码量只有700行左右。由于只需要C语言即刻，所以openwrt肯定可以用的了。

修改过的mproxy:上面说到,想要免流就必须骗过运营商,让运营商计费系统以为你流量的目的地是免流量的地址,但是原版的mproxy会根据openvpn客户端封装的remote地址将流量转发到Remote,这显然不是我们希望的!
所以,我们就需要修改版的mproxy.

流量流程如下:
手机浏览器google→→①→→openvpn-client →→②→→mproxy→→③→→openvpn-server→→④→→google

修改过的mproxy作用,就是让mproxy忽视openvpn-client的remote地址,将收到的流量全部重定向到openvpn-server上,那么我们的openvpn就建立起连接.
过程③就是其中猫腻所在,正常的mproxy会将流量在过程③中直接转发到例如wap.10086.cn.这个地址肯定不会有openvpn-server的存在,所以就一直连接失败咯.

服务器搭建

1.openwrt安装openvpn

修改opkg源,编辑/etc/opkg
以下是hg255d的源,因为hg255d的CPU是ramips的rt305x,所以我就到openwrt.org找到官方源
root@OpenWrt:/#vi /etc/opkg.conf

:wq保存
opkg update
opkg list |grep openvpn

发现已经有openvpn在list里面了,直接装.
opkg install openvpn openvpn-easy-rsa
检查一下是否安装成功
opkg list-installed |grep openvpn

OK已经安装成功了

2. 启用虚拟网卡tap0并加入br-lan网桥.(这里引用的是openwrt.org.cn,ccbcfan写的教程)

引用自[原创]openwrt下架设openvpn服务器的方法！！！
建立脚本/etc/openvpn/startupscript内容如下

修改脚本执行权限：chmod +x /etc/openvpn/startupscript
然后执行一次：
/etc/openvpn/startupscript up

就可以将tap0加入到br-lan桥中了。
root@OpenWrt# brctl show ###显示现有网桥情况，多了个tap0.

每次重启路由后，你需要重新执行一次脚本才能正常启用openvpn，你也可以修改一下将启动脚本放到/etc/init.d/，增加/etc/rc.d启动项，做到开机自动启动。更简单的办法是：将/etc/openvpn/startupscript up 加入到/etc/rc.local中开机自动启动网桥部分！！

3.生成证书

第一步中我们安装了openvpn-easy-rsa,这货就是用来生成证书用的,安装后,我们发现在/etc下有个文件夹叫easy-rsa的目录,这个就是它的工作目录了.
首先 我们需要修改/etc/easy-rsa/vars,将证书模板修改成我们自己的信息
root@OpenWrt:~# vi /etc/easy-rsa/vars

赋予vars执行权限
输入命令chmod +x /etc/easy-rsa/vars
root@OpenWrt:~# chmod +x /etc/easy-rsa/vars
执行vars
输入命令/etc/easy-rsa/vars
root@OpenWrt:~# /etc/easy-rsa/vars
NOTE: If you run ./clean-all, I will be doing a rm -rf on /etc/easy-rsa/keys
清理证书目录
root@OpenWrt:/# clean-all
NOTE: If you run ./clean-all, I will be doing a rm -rf on /etc/easy-rsa/keys
此时我们看看/etc/easy-rsa/里只有三个文件

其中keys就是我们刚刚初始化的keys目录,里面存的就是我们的证书和客户端的密匙
首先我们生成CA证书,一路回车即可,因为我们已经修改过vars模板,这里不需要修改了

此时看看keys目录已经多了个ca.crt和ca.key文件

接着我们继续,生成dh(迪菲·赫尔曼参数),这个时间会比较久,我hg255d用了差不多5分钟时间.
输入命令build-dh

查看下keys,多了个dh2048.pem文件

证明已经成功
继续生成服务器的证书输入命令build-key-server server 依然是一路回车,见到y就按y回车

再次查看keys已经发现多了很多文件,server的crt;csr;key都在了

继续生成客户端证书
输入命令 build-key client
依然是一路回车,看到y选y回车

证书已生成完毕,接下来下一步骤配置openvpn服务器
